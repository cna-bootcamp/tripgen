# TripGen UI/UX 화면 설계서

**프로젝트**: 여행 일정 생성 서비스 (TripGen)  
**작성자**: 최프론트/뷰어  
**작성일**: 2025-07-24  
**최종 수정**: 2025-07-26  
**버전**: 2.2  

## 📋 개요

본 문서는 AI 기반 여행 상세 일정 생성 서비스의 UI/UX 설계 내용을 체계적으로 정리한 화면 설계서입니다. 프로토타입 기반으로 재구성된 7개 핵심 화면의 구조, 기능, 상호작용을 상세히 기술합니다.

### 설계 범위
- **총 화면 수**: 7개 핵심 화면 + 1개 인덱스 페이지
- **대상 서비스**: 5개 마이크로서비스 (PROF, ITGEN, LOC, MAP, REV)
- **유저스토리**: 18개 요구사항 반영
- **디바이스**: 모바일(320px) ~ 데스크톱(1280px+) 반응형 지원
- **통합 기능**: 이동수단 설정(기본설정 통합), 장소 검증(장소상세정보 통합), 리뷰(장소상세정보 통합)

### 프로토타입 화면 구성
현재 구현된 프로토타입 기준:
- **01-대시보드**: 서비스 메인 진입점, 프로젝트 관리 및 주변장소검색 기능 제공
- **02-기본설정**: 여행명, 멤버 정보, 이동수단 설정
- **03-여행일정설정**: 도시별 여행 일정 및 숙소 정보 설정
- **05-AI일정생성**: AI 기반 상세 일정 생성
- **06-생성된일정조회**: 생성된 일정 확인 및 관리
- **07-장소상세정보**: 장소 정보, 리뷰, 운영시간 통합 표시
- **09-주변장소검색**: 현재 위치 기반 주변 장소 검색
- **index.html**: 프로토타입 인덱스 페이지

## 🗺️ 사용자 플로우

### 주요 사용자 여정
1. **서비스 진입** → 대시보드 (01)
2. **프로파일 설정** → 기본설정 (02) → 여행일정설정 (03)
3. **AI 일정 생성** → AI일정생성 (05) → 생성된일정조회 (06)
4. **장소 상세 확인 (선택적)** → 장소상세정보 (07)
5. **위치 서비스 (필요시)** → 주변장소검색 (09) → 이동경로확인 (10)

### 사용자 타입별 경로
- **계획형**: 1→2→3→4→5 (모든 단계 활용)
- **간편형**: 1→2→3 (핵심 기능만 사용)
- **탐색형**: 1→5 (주변장소검색 중심)

## 📱 화면별 상세 설계

### 01. 대시보드
**목적**: 서비스 진입점 및 프로젝트 관리  
**서비스**: 공통  
**우선순위**: 핵심 화면  

#### 주요 기능
- 새 여행 프로젝트 생성
- 진행 중인 프로젝트 목록 조회
- **주변장소검색 핵심 기능 노출** (신규)
- 최근 활동 및 추천 여행지 표시
- 사용자 계정 관리

#### UI 구성 요소
- **헤더**: 표준화된 네비게이션 메뉴 (4개 주요 항목: 홈, 새 여행 계획, 여행보기, 주변장소 검색)
- **히어로 섹션**: CTA 버튼 (새 여행 계획하기)
- **🆕 주변장소검색 카드**: 대형 그라데이션 카드로 핵심 기능 강조
  - 위치: 히어로 섹션 바로 아래
  - 디자인: 보라색 그라데이션 배경, 4rem 아이콘
  - 기능 설명: "주변의 맛집, 관광지, 빨래방을 찾아 보세요."
  - 호버 애니메이션 및 그림자 효과
- **여행 상태별 섹션**: 
  - 여행중: 현재 진행 중인 여행
  - 계획중: 계획 단계의 여행
  - 추억중: 완료된 여행 (기존 '갔다옴' → '추억중'으로 변경)
- **프로젝트 카드**: 진행률, 썸네일, 기본 정보, 보기/공유 버튼
- **플로팅 액션 버튼**: 우하단 2개 버튼 (여행계획 ✈️, 장소검색 📍)

#### 주변장소검색 접근 경로 (총 4개)
1. **주요 기능 카드**: 상단 대형 카드 클릭
2. **통계 섹션**: "저장된 장소" 클릭
3. **최근 활동**: "새로운 장소를 저장했습니다" 클릭
4. **플로팅 버튼**: 우하단 📍 버튼 클릭

#### 반응형 동작
- **모바일**: 1열 스택, 플로팅 CTA 버튼
- **태블릿**: 2열 그리드, 사이드바 토글
- **데스크톱**: 3열 그리드, 고정 사이드바

---

### 02. 기본설정
**목적**: 여행 기본 정보, 멤버 정보 및 이동수단 선호도 설정  
**서비스**: PROF (Travel Profile Management Service)  
**유저스토리**: RQ-PROF-001  
**우선순위**: Must Have  

#### 주요 기능
- 여행명 입력 (필수)
- 멤버 추가/삭제 (최대 10명)
- 멤버별 기본 정보 입력:
  - 이름, 나이, 성별
  - 건강상태 선택 (우수/양호/주의/제한)
  - 선호 활동 다중 선택 (관광/쇼핑/문화체험/자연체험/스포츠/휴식)
- 선호 이동수단 설정 (대중교통/자가용) - 필수 선택

#### 폼 유효성 검사
- 여행명: 필수 입력
- 멤버 이름: 필수 입력, 2-20자
- 나이: 0-120세 범위
- 성별: 라디오 버튼 선택
- 건강상태: 4단계 라디오 버튼
- 선호 활동: 다중 선택 체크박스
- 이동수단: 라디오 버튼 (필수)

#### UI 패턴
- **스테퍼 네비게이션**: 기본설정 → 여행일정설정 → AI일정생성
- **카드 레이아웃**: 멤버별 개별 카드
- **동적 추가/삭제**: "멤버 추가" 버튼으로 새 카드 생성
- **자동 저장**: LocalStorage를 통한 데이터 유지

### 03. 여행일정설정
**목적**: 여행지별 세부 일정 설정  
**서비스**: PROF (Travel Profile Management Service)  
**유저스토리**: RQ-PROF-002  
**우선순위**: Must Have  

#### 주요 기능
- 여행 계획 카드 단위로 관리
- 여행지별 기간 설정 (출발일/도착일)
- 여행지 선택 (국내/해외 도시)
- 숙소 정보 입력
- 체크인/체크아웃 시간 분리 입력

#### 입력 컴포넌트
- **카드 UI**: 여행 계획별 독립적인 카드
- **날짜 선택기**: 출발일/도착일 개별 선택
- **위치 자동완성**: 여행지 검색 및 선택
- **숙소 검색**: 숙소명 입력 및 자동완성
- **시간 입력**: 체크인/체크아웃 시간 분리 입력

#### 유효성 검사
- 종료일 > 시작일
- 출발지 ≠ 도착지 (선택사항)
- 숙소 위치 검증


### 05. AI일정생성
**목적**: Claude API 기반 맞춤형 일정 생성  
**서비스**: ITGEN (AI Itinerary Generation Service)  
**유저스토리**: RQ-ITGEN-001  
**우선순위**: Must Have (핵심)  

#### 주요 기능
- AI 일정 생성 요청
- 실시간 생성 진행률 표시
- 생성 중 취소 기능
- 재생성 옵션

#### AI 처리 플로우
1. **데이터 수집**: 프로파일 + 멤버 + 선호도
2. **API 호출**: Claude API 요청
3. **진행률 표시**: 웹소켓을 통한 실시간 업데이트
4. **결과 표시**: 생성 완료 시 자동 이동

#### 로딩 상태 관리
- **단계별 프로그레스**: 데이터 준비 → API 호출 → 결과 생성
- **예상 소요시간**: 30초-2분 가이드
- **중간 피드백**: "관광지 분석 중...", "일정 최적화 중..."

### 06. 생성된일정조회
**목적**: AI 생성 일정 조회 및 편집  
**서비스**: ITGEN (AI Itinerary Generation Service)  
**연관 화면**: 05, 07, 08  

#### 주요 기능
- 날짜별 타임라인 표시
- 시간대별 일정 상세 정보
- 드래그 앤 드롭 편집
- 일정 항목 추가/삭제

#### 타임라인 UI
- **시간 축**: 00:00-24:00 세로 타임라인
- **일정 카드**: 장소명, 소요시간, 이동시간
- **편집 모드**: 인라인 편집, 모달 상세 편집
- **저장 상태**: 자동 저장, 변경 히스토리

#### 인터랙션
- **드래그 앤 드롭**: 일정 순서 변경
- **클릭 편집**: 개별 항목 수정
- **시간 조정**: 슬라이더로 소요시간 변경

### 07. 장소상세정보
**목적**: 표준화된 장소 정보 제공  
**서비스**: ITGEN (AI Itinerary Generation Service) + REV (Review Integration Service)  
**유저스토리**: RQ-ITGEN-003, RQ-REV-001  
**우선순위**: Should Have  

#### 표준 템플릿 구성
- **기본 정보**: 장소명, 카테고리, 평점
- **위치 정보**: 정확한 도로명 주소, GPS 좌표
- **운영 정보**: 영업시간, 연락처, 웹사이트
- **편의 정보**: 주차, 접근성, 결제 방법
- **추천 이유**: AI가 제공하는 추천 근거
- **🆕 통합 리뷰 정보**: 다중 플랫폼 리뷰 요약 (12번 화면 기능 통합)
- **🆕 장소 검증 상태**: 실존 여부 및 정보 정확성 표시 (08번 화면 기능 일부 통합)

#### 시각적 요소
- **이미지 갤러리**: 장소 사진 (최대 5장)
- **지도 미리보기**: 위치 표시
- **아이콘 시스템**: 편의시설 아이콘
- **평점 시각화**: 별점 + 막대 그래프
- **🆕 리뷰 섹션**: 플랫폼별 평점, 주요 키워드, 최신 리뷰
- **🆕 검증 배지**: 정보 신뢰도 표시

### 09. 주변장소검색
**목적**: 위치 기반 장소 검색 **(핵심 기능)**  
**서비스**: LOC (Location Search Service)  
**유저스토리**: RQ-LOC-001  
**우선순위**: Must Have  
**🆕 대시보드 노출**: 상단 대형 카드로 주요 기능 강조

#### 검색 기능
- **현재 위치 감지**: GPS 기반 자동 위치 확인
- **수동 위치 입력**: 주소/좌표 직접 입력
- **검색 반경**: 500m ~ 10km 설정
- **카테고리 필터**: 음식점, 관광지, 쇼핑, 숙박 등

#### 지역별 API 연동
- **국내**: 카카오 로컬 API
- **해외**: 구글 플레이스 API
- **자동 판별**: 좌표 기반 국가 판별

#### 검색 결과 표시
- **리스트 뷰**: 거리순, 평점순 정렬
- **맵 뷰**: 지도 위 마커 표시
- **상세 정보**: 장소 정보 미리보기
