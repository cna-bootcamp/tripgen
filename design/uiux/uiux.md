# TripGen UI/UX 화면 설계서

**프로젝트**: 여행 일정 생성 서비스 (TripGen)  
**작성자**: 최프론트/뷰어  
**작성일**: 2025-07-24  
**버전**: 1.0  

## 📋 개요

본 문서는 AI 기반 여행 상세 일정 생성 서비스의 UI/UX 설계 내용을 체계적으로 정리한 화면 설계서입니다. 17개 유저스토리를 기반으로 설계된 12개 핵심 화면의 구조, 기능, 상호작용을 상세히 기술합니다.

### 설계 범위
- **총 화면 수**: 12개 핵심 화면 + 1개 인덱스 페이지
- **대상 서비스**: 5개 마이크로서비스 (PROF, ITGEN, LOC, MAP, REV)
- **유저스토리**: 17개 요구사항 반영
- **디바이스**: 모바일(320px) ~ 데스크톱(1280px+) 반응형 지원

## 🗺️ 사용자 플로우

### 주요 사용자 여정
1. **서비스 진입** → 대시보드 (01)
2. **프로파일 설정** → 멤버관리 (02) → 여행일정설정 (03) → 이동수단설정 (04)
3. **AI 일정 생성** → AI일정생성 (05) → 생성된일정조회 (06)
4. **장소 검증 (선택적)** → 장소상세정보 (07) → 장소검증결과 (08) → 리뷰평점조회 (12)
5. **위치 서비스 (필요시)** → 주변장소검색 (09) → 이동경로확인 (10) → 지도통합화면 (11)

### 사용자 타입별 경로
- **계획형**: 1→2→3→4→5 (모든 단계 활용)
- **간편형**: 1→2→3 (핵심 기능만 사용)
- **탐색형**: 1→2→3→5 (장소 검색 중심)

## 📱 화면별 상세 설계

### 01. 대시보드
**목적**: 서비스 진입점 및 프로젝트 관리  
**서비스**: 공통  
**우선순위**: 핵심 화면  

#### 주요 기능
- 새 여행 프로젝트 생성
- 진행 중인 프로젝트 목록 조회
- 최근 활동 및 추천 여행지 표시
- 사용자 계정 관리

#### UI 구성 요소
- **헤더**: 서비스 로고, 사용자 메뉴, 알림
- **히어로 섹션**: CTA 버튼 (새 여행 계획하기)
- **프로젝트 카드**: 진행률, 썸네일, 기본 정보
- **추천 섹션**: 인기 여행지, 계절별 추천

#### 반응형 동작
- **모바일**: 1열 스택, 플로팅 CTA 버튼
- **태블릿**: 2열 그리드, 사이드바 토글
- **데스크톱**: 3열 그리드, 고정 사이드바

---

### 02. 멤버관리
**목적**: 여행 멤버 정보 등록 및 관리  
**서비스**: PROF (Travel Profile Management Service)  
**유저스토리**: RQ-PROF-001  
**우선순위**: Must Have  

#### 주요 기능
- 멤버 추가/수정/삭제
- 기본 정보 입력 (이름, 나이, 성별)
- 건강상태 설정 (우수/양호/주의/제한)
- 선호도 설정 (음식, 활동, 숙박 유형)

#### 폼 유효성 검사
- 이름: 필수 입력, 2-20자
- 나이: 0-120세 범위
- 건강상태: 4단계 라디오 버튼
- 선호도: 다중 선택 체크박스

#### UI 패턴
- **스테퍼 형태**: 단계별 정보 입력
- **카드 레이아웃**: 멤버별 개별 카드
- **인라인 편집**: 클릭 시 즉시 편집 모드

---

### 03. 여행일정설정
**목적**: 기본 여행 정보 설정  
**서비스**: PROF (Travel Profile Management Service)  
**유저스토리**: RQ-PROF-002  
**우선순위**: Must Have  

#### 주요 기능
- 여행 기간 설정 (시작일/종료일)
- 출발지/도착지 선택
- 숙소 정보 입력
- 선호 관광지 카테고리 선택

#### 입력 컴포넌트
- **날짜 선택기**: 캘린더 UI, 기간 선택
- **위치 자동완성**: 도시/국가 검색
- **숙소 정보**: 주소, 체크인/아웃 시간
- **카테고리 선택**: 태그 형태 다중 선택

#### 유효성 검사
- 종료일 > 시작일
- 출발지 ≠ 도착지 (선택사항)
- 숙소 위치 검증

---

### 04. 이동수단설정
**목적**: 선호 이동수단 설정  
**서비스**: PROF (Travel Profile Management Service)  
**유저스토리**: RQ-PROF-003  
**우선순위**: Should Have  

#### 주요 기능
- 주 이동수단 선택 (대중교통/자가용)
- 세부 옵션 설정 (버스, 지하철, 택시 등)
- 지역별 선호도 차별화
- 예산 범위 설정

#### UI 컴포넌트
- **토글 스위치**: 대중교통 vs 자가용
- **체크박스 그룹**: 세부 이동수단
- **슬라이더**: 예산 범위 조절
- **지도 미리보기**: 선택된 옵션 시각화

### 05. AI일정생성
**목적**: Claude API 기반 맞춤형 일정 생성  
**서비스**: ITGEN (AI Itinerary Generation Service)  
**유저스토리**: RQ-ITGEN-001  
**우선순위**: Must Have (핵심)  

#### 주요 기능
- AI 일정 생성 요청
- 실시간 생성 진행률 표시
- 생성 중 취소 기능
- 재생성 옵션

#### AI 처리 플로우
1. **데이터 수집**: 프로파일 + 멤버 + 선호도
2. **API 호출**: Claude API 요청
3. **진행률 표시**: 웹소켓을 통한 실시간 업데이트
4. **결과 표시**: 생성 완료 시 자동 이동

#### 로딩 상태 관리
- **단계별 프로그레스**: 데이터 준비 → API 호출 → 결과 생성
- **예상 소요시간**: 30초-2분 가이드
- **중간 피드백**: "관광지 분석 중...", "일정 최적화 중..."

---

### 06. 생성된일정조회
**목적**: AI 생성 일정 조회 및 편집  
**서비스**: ITGEN (AI Itinerary Generation Service)  
**연관 화면**: 05, 07, 08  

#### 주요 기능
- 날짜별 타임라인 표시
- 시간대별 일정 상세 정보
- 드래그 앤 드롭 편집
- 일정 항목 추가/삭제

#### 타임라인 UI
- **시간 축**: 00:00-24:00 세로 타임라인
- **일정 카드**: 장소명, 소요시간, 이동시간
- **편집 모드**: 인라인 편집, 모달 상세 편집
- **저장 상태**: 자동 저장, 변경 히스토리

#### 인터랙션
- **드래그 앤 드롭**: 일정 순서 변경
- **클릭 편집**: 개별 항목 수정
- **시간 조정**: 슬라이더로 소요시간 변경

### 07. 장소상세정보
**목적**: 표준화된 장소 정보 제공  
**서비스**: ITGEN (AI Itinerary Generation Service)  
**유저스토리**: RQ-ITGEN-003  
**우선순위**: Should Have  

#### 표준 템플릿 구성
- **기본 정보**: 장소명, 카테고리, 평점
- **위치 정보**: 정확한 도로명 주소, GPS 좌표
- **운영 정보**: 영업시간, 연락처, 웹사이트
- **편의 정보**: 주차, 접근성, 결제 방법
- **추천 이유**: AI가 제공하는 추천 근거

#### 시각적 요소
- **이미지 갤러리**: 장소 사진 (최대 5장)
- **지도 미리보기**: 위치 표시
- **아이콘 시스템**: 편의시설 아이콘
- **평점 시각화**: 별점 + 막대 그래프

### 08. 장소검증결과
**목적**: AI 추천 장소의 실존성 검증  
**서비스**: ITGEN (AI Itinerary Generation Service)  
**유저스토리**: RQ-ITGEN-002  
**우선순위**: Must Have  

#### 검증 항목
- **실존 여부**: 지도 API 기반 확인
- **정보 정확성**: 주소, 연락처, 영업시간
- **접근성**: 대중교통, 주차 가능 여부
- **신뢰도 점수**: 0-100점 종합 평가

#### 검증 상태 표시
- ✅ **검증 완료**: 모든 정보 확인됨
- ⚠️ **부분 검증**: 일부 정보 불확실
- ❌ **검증 실패**: 정보 불일치 또는 미존재
- 🔄 **검증 중**: 실시간 검증 진행

#### 대안 제안
- 검증 실패 시 유사 장소 추천
- 거리 기반 대체 장소 목록
- 사용자 직접 장소 추가 옵션

---

### 09. 주변장소검색
**목적**: 위치 기반 장소 검색  
**서비스**: LOC (Location Search Service)  
**유저스토리**: RQ-LOC-001  
**우선순위**: Must Have  

#### 검색 기능
- **현재 위치 감지**: GPS 기반 자동 위치 확인
- **수동 위치 입력**: 주소/좌표 직접 입력
- **검색 반경**: 500m ~ 10km 설정
- **카테고리 필터**: 음식점, 관광지, 쇼핑, 숙박 등

#### 지역별 API 연동
- **국내**: 카카오 로컬 API
- **해외**: 구글 플레이스 API
- **자동 판별**: 좌표 기반 국가 판별

#### 검색 결과 표시
- **리스트 뷰**: 거리순, 평점순 정렬
- **맵 뷰**: 지도 위 마커 표시
- **상세 정보**: 장소 정보 미리보기

---

### 10. 이동경로확인
**목적**: 이동수단별 경로 및 시간 계산  
**서비스**: LOC (Location Search Service)  
**유저스토리**: RQ-LOC-002  
**우선순위**: Must Have  

#### 경로 계산
- **대중교통**: 버스, 지하철, 환승 정보
- **자가용**: 최단/최적 경로, 실시간 교통
- **도보**: 도보 경로, 예상 소요시간
- **복합 이동**: 여러 수단 조합 경로

#### 시간 필터링
- **대중교통/자가용**: 60분 이내
- **도보**: 20분 이내
- **사용자 설정**: 커스텀 시간 범위

#### 경로 정보 표시
- **지도 오버레이**: 경로 라인 표시
- **단계별 안내**: 상세 이동 가이드
- **비용 정보**: 교통비, 주차비, 통행료
- **실시간 정보**: 교통 상황, 지연 정보

---

### 11. 지도통합화면
**목적**: 통합 지도 서비스 제공  
**서비스**: MAP (Map Integration Service)  
**유저스토리**: RQ-MAP-001  
**우선순위**: Must Have  

#### 지도 API 통합
- **지역별 자동 선택**: 국내(카카오맵) / 해외(구글맵)
- **일관된 인터페이스**: 통합된 UI/UX
- **데이터 표준화**: 공통 데이터 포맷
- **폴백 처리**: API 장애 시 대체 서비스

#### 주요 기능
- **전체 여행 경로**: 일정별 연결된 경로 표시
- **레이어 관리**: 숙소, 관광지, 음식점 등 카테고리별 표시
- **인터랙티브**: 마커 클릭 시 상세 정보
- **경로 최적화**: 효율적인 이동 순서 제안

#### 사용자 인터페이스
- **지도 컨트롤**: 확대/축소, 레이어 토글
- **정보 패널**: 선택된 장소 상세 정보
- **경로 패널**: 이동 경로 및 시간 정보

---

### 12. 리뷰평점조회
**목적**: 다중 플랫폼 리뷰 통합 조회  
**서비스**: REV (Review Integration Service)  
**유저스토리**: RQ-REV-001  
**우선순위**: Should Have  

#### 리뷰 수집 소스
- **구글 리뷰**: Google Places API
- **네이버 리뷰**: 네이버 플레이스 API
- **트립어드바이저**: TripAdvisor API
- **기타**: 망고플레이트, 포스퀘어 등

#### 통합 평점 시스템
- **가중 평균**: 플랫폼별 신뢰도 반영
- **평점 분포**: 5점 만점 히스토그램
- **최신성**: 최근 6개월 리뷰 우선
- **필터링**: 언어별, 평점별, 날짜별

#### 리뷰 표시
- **요약 카드**: 평점, 리뷰 수, 키워드
- **상세 리뷰**: 원본 텍스트, 번역 옵션
- **감정 분석**: 긍정/부정 비율
- **추천 여부**: AI 기반 추천 점수

## 🔄 화면 간 전환 및 네비게이션

### 네비게이션 패턴
- **순차 진행**: 프로파일 설정 단계 (02→03→04)
- **허브 방식**: 대시보드를 중심으로 한 방사형 이동
