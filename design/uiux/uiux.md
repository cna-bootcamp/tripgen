# 여행 일정 생성 서비스 - UI/UX 설계서

## 프로토타입 화면 목록 정의

### 인증 영역
1. **01-로그인화면** - 사용자 인증
2. **02-회원가입화면** - 신규 사용자 등록

### 메인 영역  
3. **03-대시보드** - 홈 화면, 주요 서비스 접근점
4. **04-프로필관리** - 사용자 정보 수정

### 여행 계획 프로세스 (순차적)
5. **05-기본설정** - 여행명, 멤버, 이동수단 설정
6. **06-여행지설정** - 여행 일정 및 숙소 관리
7. **07-AI일정생성** - AI 일정 생성 요청 및 진행 상황
8. **08-생성된일정조회** - 일자별 상세 일정 관리

### 장소 관련 기능
9. **09-주변장소검색** - 위치 기반 장소 검색
10. **10-장소상세정보** - 장소 상세 정보 및 AI 추천

### 여행 관리
11. **11-여행목록전체보기** - 전체 여행 목록 관리

### 모바일 전용
12. **12-햄버거메뉴** - 모바일 네비게이션 메뉴

## 화면 간 사용자 플로우 정의

### 주요 사용자 여정

```
[신규 사용자 여정]
회원가입 → 로그인 → 대시보드 → 새 여행 계획 → 기본설정 → 여행지설정 → AI일정생성 → 생성된일정조회

[기존 사용자 여정] 
로그인 → 대시보드 → (여행 카드 선택) → 생성된일정조회 → 일정 관리/수정

[장소 탐색 여정]
대시보드 → 주변장소검색 → 장소상세정보 → (길찾기/공유)

[여행 관리 여정]
대시보드 → 여행목록전체보기 → (필터링/검색) → 여행 선택 → 생성된일정조회
```

### 네비게이션 구조

**허브형 구조**: 대시보드를 중심으로 한 방사형 네비게이션
- 대시보드 ↔ 모든 주요 기능
- 여행 계획 프로세스: 선형 진행 (4단계)
- 모바일: 하단 네비게이션 바 + 햄버거 메뉴

## 화면별 상세 설계

### 03-대시보드
**개요**: 서비스의 메인 허브, 사용자의 여행 현황과 주요 기능 접근점
**관련 유저스토리**: UFR-TRIP-010, UFR-TRIP-020
**비즈니스 중요도**: 높음 (서비스 첫인상)

**주요 기능**:
- 주요 서비스 빠른 접근 (배너)
- 여행 상태별 관리 (여행중/계획중/추억중)
- 사용자 프로필 표시

**UI 구성요소**:
- 헤더: 로고, 사용자 아바타/드롭다운
- 주요 서비스 배너 (2개): 새 여행 계획하기, 주변 장소 검색
- 여행 상태별 섹션 (3개): 각각 최대 3개 카드 표시
- 하단 네비게이션 바 (모바일)

**인터랙션**:
- 배너 클릭 → 해당 기능 화면 이동
- 여행 카드 클릭 → 생성된일정조회 화면 이동
- "전체보기" 클릭 → 여행목록전체보기 화면 이동

---

### 05-기본설정
**개요**: 새 여행 계획의 첫 단계, 여행 기본 정보 설정
**관련 유저스토리**: UFR-TRIP-030
**비즈니스 중요도**: 높음 (여행 계획 시작점)

**주요 기능**:
- 여행명 및 이동수단 설정
- 멤버 정보 관리 (동적 추가/수정/삭제)
- 실시간 입력 검증 및 자동저장

**UI 구성요소**:
- 상단: 진행바 (1/4 단계)
- 여행명 입력: 최대 16자
- 이동수단 선택: 라디오 버튼 (대중교통/자동차)
- 멤버 관리 영역: 카드 형태, '+' 버튼으로 추가
- 하단: 이전/다음 버튼

**인터랙션**:
- 멤버 카드 선택 → 편집/삭제 버튼 표시
- 실시간 자동저장 (3초 debounce)
- 다음 버튼 → 필수 정보 검증 후 여행지설정 화면 이동

---

### 08-생성된일정조회
**개요**: AI가 생성한 여행 일정을 조회하고 관리하는 핵심 화면
**관련 유저스토리**: UFR-TRIP-060, UFR-TRIP-070, UFR-TRIP-080
**비즈니스 중요도**: 최고 (서비스 핵심 가치)

**주요 기능**:
- 일자별 탭 구분으로 일정 조회
- 장소별 상세 정보 및 이동 경로 표시
- 드래그앤드롭으로 순서 변경
- 일정 재생성 및 내보내기

**UI 구성요소**:
- 상단 액션 버튼: 🔄 일정 재생성, 📥 내보내기
- 일자별 탭: 차수, 날짜, 도시명, 날씨 아이콘
- 장소 카드: 시간, 장소명, 카테고리, 소요시간
- 이동 정보 카드: 수단 아이콘, 소요시간, 경로
- 건강상태 고려 정보: 휴식 포인트, 접근성 아이콘

**인터랙션**:
- 탭 선택 → 해당일 일정 표시, "🔄 다시 생성" 버튼 표시
- 장소 카드 선택 → 상세정보/길찾기/삭제 버튼 표시
- 드래그앤드롭 → 실시간 순서 변경 및 자동저장
- 일정 재생성 → 해당일만 새로 생성

---

### 09-주변장소검색
**개요**: 위치 기반으로 주변 장소를 검색하고 필터링
**관련 유저스토리**: UFR-LOC-010, UFR-LOC-020
**비즈니스 중요도**: 중간 (부가 기능)

**주요 기능**:
- 출발지, 이동수단, 시간 범위 설정
- 카테고리별 필터링
- 키워드 검색 및 무한 스크롤

**UI 구성요소**:
- 검색 조건 영역: 출발지, 이동수단, 시간 범위
- 키워드 검색창: "✨ 찾을 장소 종류/이름 입력"
- 카테고리 필터: 전체/관광지/맛집/빨래방 (라디오 버튼)
- 검색 결과: 장소 카드 목록
- 정렬 옵션: 시간순/평점순/이름순/리뷰순

**인터랙션**:
- 검색 조건 변경 → 자동 검색 실행
- 카테고리 선택 → 해당 카테고리만 필터링
- 무한 스크롤 → 추가 결과 자동 로드
- 장소 카드 클릭 → 장소상세정보 화면 이동

---

### 12-햄버거메뉴 (모바일 전용)
**개요**: 모바일에서 전체 기능에 접근할 수 있는 네비게이션 메뉴
**관련 유저스토리**: UFR-TRIP-022
**비즈니스 중요도**: 중간 (모바일 UX)

**주요 기능**:
- 전체 화면 메뉴 패널
- 주요 기능 빠른 접근
- 현재 페이지 표시

**UI 구성요소**:
- 전체 화면 오버레이 (흰색 배경)
- 메뉴 항목 (세로 목록):
  - 🏠 홈 (대시보드)
  - ✈️ 새 여행 (기본설정)
  - 📍 장소검색 (주변장소검색)
  - 📋 여행목록 (여행목록전체보기)

**인터랙션**:
- 햄버거 버튼(☰) 탭 → 메뉴 패널 표시
- 메뉴 외부 영역 탭 → 메뉴 닫힘
- 메뉴 항목 탭 → 해당 화면 이동, 메뉴 자동 닫힘
- 현재 페이지 → 파란색 강조, 클릭 불가

## 화면간 전환 및 네비게이션

### 데스크톱 네비게이션
- **상단 헤더**: 로고(홈 링크), 주요 메뉴, 사용자 프로필
- **브레드크럼**: 현재 위치 표시 (여행 계획 프로세스)
- **사이드바**: 필요시 보조 네비게이션

### 모바일 네비게이션
- **하단 네비게이션 바**: 주요 4개 기능 고정
  - 🏠 홈, ✈️ 새 여행, 📍 장소검색, 📋 여행목록
- **햄버거 메뉴**: 전체 기능 접근
- **진행바**: 여행 계획 프로세스 단계 표시

### 화면 전환 패턴
- **페이드**: 기본 화면 전환
- **슬라이드**: 단계형 프로세스 (좌우)
- **모달**: 삭제 확인, 로그아웃 등
- **드로어**: 햄버거 메뉴 (하단에서 위로)

## 반응형 설계 전략

### 브레이크포인트
- **Mobile**: 320px ~ 767px (주 타겟)
- **Tablet**: 768px ~ 1023px
- **Desktop**: 1024px 이상

### Mobile First 접근
모든 기능이 모바일에서 완전히 작동하도록 설계하고, 큰 화면으로 확장

### 적응형 레이아웃

**대시보드**:
- Mobile: 주요 배너 세로 배치, 여행 카드 1열
- Tablet: 주요 배너 가로 배치, 여행 카드 2열  
- Desktop: 주요 배너 가로 배치, 여행 카드 3열

**여행 계획 프로세스**:
- Mobile: 단일 컬럼, 하단 네비게이션
- Desktop: 진행바 상단, 넓은 폼 레이아웃

**장소 검색**:
- Mobile: 스택형 레이아웃
- Desktop: 사이드바 필터 + 메인 결과

### 터치 최적화
- 최소 터치 영역: 48px × 48px
- 버튼 간 충분한 간격: 8px 이상
- 스와이프/드래그 제스처 지원

## 접근성 보장 방안

### WCAG 2.1 AA 준수
- **색상 대비비**: 4.5:1 이상 (일반 텍스트), 3:1 이상 (대형 텍스트)
- **색상 독립성**: 색상만으로 정보 구분 금지
- **키보드 네비게이션**: 모든 기능 키보드로 접근 가능
- **포커스 표시**: 명확한 포커스 인디케이터

### 시각적 접근성
- **충분한 여백**: 읽기 쉬운 레이아웃
- **명확한 계층**: 제목, 본문 구분
- **아이콘과 텍스트**: 의미 전달 보조
- **확대 지원**: 200%까지 확대 시 가독성 유지

### 스크린 리더 지원
- **ARIA 라벨**: 모든 인터랙티브 요소
- **랜드마크**: 페이지 구조 명확화
- **상태 알림**: 동적 콘텐츠 변경 시
- **건너뛰기 링크**: 메인 콘텐츠 빠른 접근

### 건강상태 고려 (유저스토리 반영)
- **휴식 포인트**: 명확한 표시와 설명
- **접근성 정보**: 엘리베이터, 경사로 표시
- **선택적 활동**: 기본/활동적 코스 구분
- **도보 거리**: 제한 멤버를 위한 안내

## 성능 최적화 방안

### 로딩 성능
- **이미지 최적화**: WebP 포맷, 적응형 이미지 크기
- **지연 로딩**: 뷰포트 밖 콘텐츠 지연 로드
- **코드 분할**: 라우트별 청크 분리
- **프리로딩**: 주요 리소스 우선 로드

### 런타임 성능
- **가상 스크롤**: 긴 목록 최적화
- **메모이제이션**: 불필요한 재렌더링 방지
- **디바운싱**: 검색 입력 최적화
- **드래그앤드롭**: 60fps 유지

### 네트워크 최적화
- **API 캐싱**: 자주 사용하는 데이터
- **오프라인 지원**: 중요 기능 오프라인 사용
- **배치 요청**: 관련 데이터 일괄 처리
- **압축**: gzip/brotli 압축 적용

### 성능 지표 목표
- **첫 콘텐츠 페인트**: 1.5초 이내
- **상호 작용까지 시간**: 3초 이내
- **누적 레이아웃 이동**: 0.1 이하
- **일정 생성 완료**: 5초 이내 (유저스토리 요구사항)
