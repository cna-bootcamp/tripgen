# 여행 상세 일정 생성 서비스 - 유저스토리

## 마이크로서비스 구성
1. **User** - 사용자 인증 및 사용자 정보 관리
2. **Trip** - 여행 기본정보(여행명, 이동수단, 여행멤버), 일간(Daily) 일정(여행 시작일, 숙소, 숙박일, 체크인/체크아웃 시간) 관리, 시간별 상세일정 요청/저장 
3. **Schedule** - AI 기반 시간별 상세 일정 생성
4. **Location** - 장소 검색

---

## 유저스토리 
```
1. User
1) 사용자 인증 및 관리
- UFR-USER-010: [회원가입] 사용자로서 | 나는, 여행 계획을 관리하기 위해 | 간편하게 회원가입하고 싶다.
- 시나리오: 회원가입 
  미 로그인 상태로 서비스에 접근한 상황에서 | 사용자 기본정보(이름, 이메일, 연락처), ID, 암호를 입력하여 회원가입 요청하면 | 회원가입이 된다.
  - 이름, 이메일, 연락처 등록 체크   
  - ID는 5자 이상의 영숫자 
  - 암호는 8자 이상의 영숫자와 특수문자가 최소 1개 이상 포함 
- M/5 

- UFR-USER-020: [로그인] 사용자로서 | 나는, 여행 계획을 관리하기 위해 | 간편하게 로그인하고 싶습니다.
- 시나리오: 로그인 
  미 로그인 상태로 서비스에 접근하여 | ID와 암호를 입력하여 로그인 요청하면 | 대시보드 페이지가 표시된다.
  - 정상 로그인 
  - ID 또는 암호가 틀린 경우 안내 메시지 표시 
- 시나리오: 프로파일 표시 
  대시보드 페이지에 접근한 상황에서 | 헤더의 사용자 아바타를 확인하면 | 로그인된 사용자의 이름 첫 글자가 표시된다.
  - 사용자 아바타 표시 (이름 첫 글자)
  - 클릭 시 사용자 기본 정보 표시: 사용자명, 이메일, 연락처  
- M/5

2. Trip
1) 여행 프로젝트 관리 
- UFR-TRIP-001: [대시보드] 여행자로서, 나는 여행 계획을 효율적으로 관리하기 위해, 대시보드에서 여행중/계획중/추억중 상태별로 내 여행들을 확인하고 관리하고 싶습니다.
- 시나리오: 대시보드에서 여행 프로젝트 보기 
  대시보드에 접근한 상황에서 | 여행 프로젝트 카드들을 확인하면 | 여행중/계획중/추억중 섹션별로 여행들이 표시되고 클릭하여 상세 조회 가능하다.
  - 여행중/계획중/추억중 3개 섹션 구분
  - 프로젝트 카드에 여행 프로젝트 대표 이미지 표시
  - 여행 제목 및 날짜 표시
  - 진행률 표시 (진행중인 여행)
  - 카드 클릭 시 상세 일정 이동
  - 새 여행 시작 버튼
  - 여행 상태 자동 업데이트
  - 최대 3개 프로젝트 표시
- M/8

2) 기본설정 관리
- UFR-TRIP-002: [기본설정] 여행자로서, 나는 여행 계획을 시작하기 위해, 여행명과 함께 여행 멤버의 기본 정보(이름, 나이, 성별, 건강상태, 선호도)와 이동수단 선호도를 한 화면에서 설정하고 싶습니다.
- 시나리오: 기본설정 화면에서 여행 정보 설정
  기본설정 화면에 접근한 상황에서 | 여행명 입력, 멤버 추가, 이동수단 선택 후 다음 단계 클릭하면 | 모든 정보가 저장되고 다음 화면으로 이동된다.
  - 여행명 필수 입력 검증
  - 여행명 자동 저장 (LocalStorage)
  - "멤버 추가" 버튼으로 새 멤버 추가
  - 멤버 카드별 정보 입력 (이름, 나이, 성별)
  - 건강상태 선택 (우수/양호/주의/제한)
  - 선호 활동 다중 선택 (관광/쇼핑/문화체험/자연체험/스포츠/휴식)
  - 멤버별 편집/삭제 기능
  - 최대 10명까지 등록 가능
  - 대중교통/자가용 선택 (라디오 버튼)
  - 자가용 기본 선택
  - 선택 필수 검증
  - 폼 데이터 자동 저장
  - 페이지 새로고침 시 데이터 복원
- M/10

3) 여행일정 설정
- UFR-TRIP-003: [여행일정설정] 여행자로서, 나는 여행 계획을 세우기 위해, 각 도시별 체류 기간과 숙소 정보를 관리하고 싶습니다.
- 시나리오: 여행 계획 카드 기반 일정 설정
  여행일정설정 화면에 접근한 상황에서 | 여행 계획 카드에서 도시, 출발일/도착일, 숙소 정보를 입력하면 | 각 여행지별 일정이 카드 형태로 저장되고 관리된다.
  - 카드별 독립적인 여행 계획 관리
  - "여행 계획 추가" 버튼으로 새 카드 생성
  - 카드별 삭제 기능 (X 버튼)
  - 여행지(도시) 입력 필드
  - 출발일/도착일 개별 날짜 선택기
  - 숙소명 입력 필드
  - 체크인/체크아웃 시간 입력
  - 도착일 > 출발일 유효성 검사
  - 체류 기간 자동 계산 및 표시
  - 폼 데이터 자동 저장
- M/8

4) 생성된 일정 관리
- UFR-TRIP-004: [생성된일정조회] 여행자로서, 나는 생성된 일정을 확인하고 수정하기 위해, 일자별로 구분된 상세 일정을 조회하고 장소를 관리하고 싶습니다.
- 시나리오: 생성된 일정 조회 및 관리
  AI가 생성한 일정이 있는 상황에서 | 생성된일정조회 화면에 접근하면 | 일자별 탭으로 구분된 일정이 표시되고 장소별 관리가 가능하다.
  - 일자별 탭 구분 (Day 1, Day 2...)
  - 시간대별 장소 카드 표시
  - 장소명, 카테고리, 소요시간 표시
  - 이동시간 및 거리 표시
  - 장소 상세정보 보기 버튼
  - 길찾기 버튼 (구글맵 연동)
  - 장소 삭제 기능
  - 일자별 재생성 기능
  - 일정 자동 저장
  - PDF 내보내기 기능
- M/13

3. AI 서비스
1) AI 일정 생성
- UFR-AI-001: [AI일정생성] 여행자로서, 나는 맞춤형 여행 일정을 얻기 위해, AI를 활용하여 여행 프로파일과 멤버 정보를 기반으로 상세 일정을 생성받고 싶습니다.
- 시나리오: AI 기반 상세 일정 생성
  여행 기본정보와 일정이 설정된 상황에서 | AI 일정 생성 화면에서 생성 시작하면 | 진행률과 함께 AI가 일정을 생성하여 표시한다.
  - 사용자 요청사항 입력 (선택)
  - 첫 일정 시작시간 설정
  - 생성 진행률 표시 (0~100%)
  - 생성 중 취소 기능
  - 멤버 건강상태 고려
  - 멤버 선호도 반영
  - 이동수단 선호도 반영
  - 숙소 위치 기반 일정 구성
  - 장소별 적정 소요시간 계산
  - 시간대별 일정 구성
  - 이동경로 최적화
  - 재생성 기능
- M/21

2) AI 추천 이유
- UFR-AI-002: [AI추천이유] 여행자로서, 나는 AI가 추천한 장소를 이해하기 위해, 각 장소별로 추천 이유와 방문 팁을 확인하고 싶습니다.
- 시나리오: AI 추천 이유 제공
  AI가 추천한 장소가 있는 상황에서 | 장소 상세정보를 확인하면 | AI 추천 이유와 방문 팁이 표시된다.
  - 장소별 AI 추천 이유 텍스트
  - 멤버 선호도와의 연관성 설명
  - 추천 방문 시간
  - 예상 소요 시간
  - 베스트 포토 스팟
  - 주의사항
  - 근처 추천 장소
- S/8

4. Location 서비스
1) 주변 장소 검색
- UFR-LOC-001: [주변장소검색접근] 여행자로서, 나는 현재 위치 주변의 관심 장소를 쉽게 찾기 위해, 대시보드의 주변장소검색 카드를 통해 검색 기능에 빠르게 접근하고 싶습니다.
- 시나리오: 대시보드에서 주변장소검색 접근
  대시보드에 접근한 상황에서 | 상단의 주변장소검색 카드 클릭하면 | 주변장소검색 화면으로 이동한다.
  - 주변장소검색 전용 카드 (그라데이션)
  - 기능 태그 표시 (맛집, 숙소, 관광지, 카페)
  - 호버 효과 및 애니메이션
  - 카드 클릭 시 즉시 이동
  - 현재 위치 자동 감지
- M/5

2) 장소 카테고리 검색
- UFR-LOC-002: [카테고리검색] 여행자로서, 나는 원하는 종류의 장소를 찾기 위해, 거리/시간 기반으로 필터링하고 카테고리별로 검색하고 싶습니다.
- 시나리오: 카테고리 및 거리 기반 검색
  주변장소검색 화면에 접근한 상황에서 | 카테고리와 거리 필터를 설정하고 검색하면 | 조건에 맞는 장소 목록이 카드 형태로 표시된다.
  - 거리 필터 (500m, 1km, 3km, 5km)
  - 시간 필터 (대중교통/차량 10~60분, 도보 5~20분)
  - 카테고리 필터 (관광지, 식당, 카페, 세탁소, 편의점)
  - 장소 카드 그리드 표시
  - 장소명, 카테고리, 거리, 평점 표시
  - 이미지 썸네일
  - 영업 상태 표시
  - 카드 클릭 시 상세정보 이동
  - 북마크 기능
- M/10

3) 장소 상세정보
- UFR-LOC-003: [장소상세정보] 여행자로서, 나는 추천된 장소의 상세 정보를 확인하기 위해, 위치 정보, 운영 정보, AI 추천 이유를 한 화면에서 확인하고 싶습니다.
- 시나리오: 장소 상세정보 통합 조회
  특정 장소를 선택한 상황에서 | 장소 상세정보 화면으로 이동하면 | 모든 정보가 통합되어 표시된다.
  - 대표 이미지 (히어로 이미지)
  - 장소명, 카테고리, 평점
  - 공유, 북마크, 길찾기 버튼
  - 정확한 주소
  - 맵 검색어
  - 주차장 검색어
  - 영업 상태 (영업중/휴무)
  - 영업 시간
  - 특별 운영 시간 (시계탑 공연 등)
  - AI 추천 이유
  - 추천 방문 시간
  - 예상 소요 시간
  - 베스트 포토 스팟
- M/8

5. Review 서비스
1) 리뷰 및 평점 표시
- UFR-REV-001: [리뷰평점표시] 여행자로서, 나는 신뢰할 수 있는 장소 선택을 위해, 장소 상세정보 화면에서 리뷰와 평점을 확인하고 싶습니다.
- 시나리오: 장소별 리뷰 및 평점 확인
  장소 상세정보 화면에 접근한 상황에서 | 리뷰 섹션을 확인하면 | 평점과 최신 리뷰들이 표시된다.
  - 전체 평점 (5점 만점)
  - 별점 시각화
  - 총 리뷰 수
  - 최신 리뷰 2-3개 표시
  - 리뷰어 이름, 날짜
  - 리뷰 내용
  - 개별 별점
  - 리뷰 더보기 버튼
  - 리뷰 정렬 옵션
- S/5

2) 리뷰 통합 관리
- UFR-REV-002: [리뷰통합관리] 시스템 관리자로서, 나는 다양한 플랫폼의 리뷰를 통합하기 위해, 구글, 네이버 등의 리뷰를 수집하고 표준화하고 싶습니다.
- 시나리오: 멀티 플랫폼 리뷰 통합
  외부 플랫폼에 리뷰가 존재하는 상황에서 | 리뷰 수집 프로세스 실행하면 | 여러 플랫폼의 리뷰가 통합되어 저장된다.
  - 구글 리뷰 API 연동
  - 네이버 리뷰 크롤링
  - 리뷰 데이터 표준화
  - 중복 리뷰 제거
  - 리뷰 신뢰도 검증
  - 정기적 업데이트
- C/13
```
